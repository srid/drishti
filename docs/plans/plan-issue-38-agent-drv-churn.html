Stop the agent drv churning on every rebuild

drishti #38 Every reconnect after a drishti rebuild re-copies + rebuilds the full agent closure on each remote. Root cause: the agent .drv is fused to the whole app build. Fix: give the agent its own workspace package so client/server edits stop rotating its hash.

Status — implemented (PR for #38). The split landed as packages/common (drishti-common, the wire contract) + packages/agent (drishti-agent), with nix/packages/drishti-agent/default.nix as the scoped build and a just drv-stability CI guard. One refinement to the plan below: the workspace split alone is not sufficient. fetchBunDeps is a symlinkJoin over every bun.nix entry, including "drishti-app" = copyPathToStore ./packages/app — so the agent's dep cache still took the app's source as a direct input. The fix pairs the split with a filtered cache: the agent derivation passes a bun.nix with the drishti-app entry removed (removeAttrs) to fetchBunDeps. Both edges (narrow src + filtered cache) are required; either alone leaves the churn intact. The single bun.lock is preserved.

1. Root cause — the agent drv is the whole app build

drishti-agent (root default.nix:48-56) is just a makeWrapper that bakes the literal store path ${drishtiBuilt}/lib/drishti/packages/app/src/agent/main.ts as bun’s launch arg. That ${drishtiBuilt} interpolation is a real closure edge — confirmed at the .drv level: nix-store --query --references on drishti-agent.drv lists drishti-built-0.1.0.drv (and bun) as its direct build inputs. So the agent’s entire build + runtime closure is drishtiBuilt.

And drishtiBuilt (nix/packages/drishti/default.nix) is a single stdenv.mkDerivation whose src fileset is the whole app tree — all of packages/app (client + server + common + agent, including every *.test.ts) plus package.json / bun.lock / bunfig.toml / tsconfig.base.json / bun.nix / scripts — and whose buildPhase emits the client dist bundle. Any edit anywhere in that tree rehashes src → new drishtiBuilt.drv → new drishti-agent.drv → new agentDrvBySystem JSON (flake.nix:43-47). The monitor then hands every remote a never-realised drv and the first reconnect pays a full cross-arch nix copy --derivation + nix-store --realise (observed ~10 min, 5 paths).

Proven, not just argued. Append a comment to packages/app/src/client/theme.ts — a file the agent never imports — and nix eval .#drishti-agent.drvPath flips (8xhfzhld… → 9m01szcb…); git checkout restores it byte-for-byte. The agent’s true source closure is 3 files: agent/main.ts, agent/proc.ts, common/surface.ts (surface.ts has no internal imports; metrics.ts/history.ts are not in the agent graph). Everything else is accidental coupling.

This is a textbook volatility leak: the agent (must stay byte-stable on remotes, changes rarely) is structurally fused to the client + server (churns on every dev edit). The fix is a decomposition, not a tweak.

2. The trap — narrowing the fileset is not enough

The obvious move — “give the agent a scoped derivation with a 3-file src fileset” — does not work, and was empirically falsified by every adversarial check. There is a second churn edge hiding in the dep cache: 

# bun.nix:415
"drishti-app" = copyPathToStore ./packages/app;
The load-bearing detail: bun2nix bakes the entire packages/app tree into a fixed-output derivation (bun-pkg-drishti-app) inside the shared bun cache that fetchBunDeps returns. So even with a 3-file src, a client edit still rotates that FOD → still churns the agent drv. Verified on a committed tree with --no-eval-cache: a client-only and a server-only edit both still moved the agent drvPath, with the sole differing input being bun-pkg-drishti-app. Narrowing src is necessary but inert while this edge is uncut.

(An early “nondeterminism” scare during analysis was a busy eval-cache SQLite artifact — a red herring. The real defect is this bunDeps FOD. Always verify through the flake, on a committed tree, with --no-eval-cache.)

3. The fix — make the agent its own workspace package

The workspace glob is packages/* with exactly one member today (packages/app). Add members so the agent’s copyPathToStore is naturally agent-only — then bun2nix emits a second local FOD whose source doesn’t move when client/server files change.

New memberContentsWhy
packages/common src/surface.ts + a tiny package.json (deps: @kolu/surface, zod) The one shared wire contract, single-owned, consumed by both app and agent. Keeps the 3-tier contract DRY and gives it its own minimal FOD.
packages/agent src/main.ts, src/proc.ts, package.json (deps: common + @orpc/server + zod + @kolu/surface) Agent-only source. Its copyPathToStore FOD is invariant to client/server edits.
Must not: leave surface.ts in packages/app. If the agent member depends back on the app member, the FOD re-fuses and the split is defeated. Extract packages/common first.

The concrete edits

  1. Extract packages/common — move surface.ts there with its own package.json; update packages/app imports to point at it.
  2. Add packages/agent — move main.ts + proc.ts there with a narrow package.json.
  3. Regenerate bun.nix — nix run .#bun2nix. Now there is a second, agent-only local FOD.
  4. New nix/packages/drishti-agent/default.nix — an stdenv.mkDerivation over the agent member:
    • No buildPhase / dist (dontBuild + dontUseBunBuild) — dropping the dist bundle is the single cleanest churn-severing move; the agent never reads it.
    • Keep bun2nix.hook + fetchBunDeps, --linker=hoisted, and the @kolu/surface hydration in postBunNodeModulesInstallPhase.
    • Install-guard only agent/main.ts; drop the server/main.ts + dist/index.html guards.
  5. Repoint the wrapper in root default.nix:54-55: 
    makeWrapper ${bun}/bin/bun $out/bin/drishti-agent \
  --add-flags "${drishtiBuilt}/lib/drishti/packages/app/src/agent/main.ts"
  --add-flags "${drishtiAgentBuilt}/lib/drishti/packages/agent/src/main.ts"
    drishtiBuilt, drishti-client, and the monitor are untouched — they still need the full tree + dist.
  6. Keep flake.nix shape — agentDrvBySystem still reads .drishti-agent.drvPath via unsafeDiscardStringContext. Fileset stays a static lib.fileset.unions of literal paths (no IFD / readDir / build-output reads) so cross-arch pure eval still emits the linux drv from a macOS evaluator.

node_modules — the unavoidable floor

The agent build still needs @kolu/surface’s full transitive closure hoisted, because bunfig.toml forces linker="hoisted" and the kolu hydration is cp -rL (not a symlink). @kolu/surface’s own package.json hard-depends on solid-js + @solid-primitives/scheduled, so those land in the agent’s node_modules even though the agent executes zero Solid. You do still drop babel, @tailwindcss/cli (+ its native oxide/watcher blobs across 12 platform variants), hono, partysocket, cleye — a real size win on the rare cold copy. The minimum is “the agent’s @orpc/zod set plus @kolu/surface’s dependency closure,” not the agent’s own imports alone.

4. The acceptance test

This is the headline criterion. It must run through the flake, on a committed tree, with --no-eval-cache (a filesystem-only fileset probe falsely passes): 

BASE=$(nix eval --raw --no-eval-cache .#packages.x86_64-linux.drishti-agent.drvPath)
printf '\n// churn-test\n' >> packages/app/src/client/theme.ts && git add -A && git commit -q -m churn-test
AFTER=$(nix eval --raw --no-eval-cache .#packages.x86_64-linux.drishti-agent.drvPath)
git reset --hard -q HEAD~1
[ "$BASE" = "$AFTER" ] && echo PASS || echo "FAIL: $BASE -> $AFTER"

PASS (byte-identical drvPath across a committed client edit) is the goal. The naive fileset-only approach prints FAIL. Land this one-liner as a CI regression guard.

6. Residual cold cases (correct, by design)

After #2, the agent drv legitimately still changes — and should — when:

  • An edit to agent/main.ts, agent/proc.ts, or common/surface.ts (the wire contract genuinely changed).
  • A bun toolchain bump or a @kolu/surface bump.
  • Any bun.lock / bun.nix change — including a client-only dep bump (e.g. tailwind) that rotates the shared lockfile. The agent member’s package.json is narrow, but the shared lockfile is still an input. Eliminating this entirely would need a fully independent agent lockfile; the rot/maintenance cost isn’t worth it.

These residuals are exactly where direction 1 (pre-warm) and direction 4 (Tier A “may take minutes” framing) earn their keep. The GC-root pin (#37) is orthogonal and stays — it can’t help a changed drv, but it keeps the unchanged-drv path warm across remote GC.

8. Implementation caveats

  • Single-own the wire contract. surface.ts lives in packages/common; the agent must not depend back on packages/app (re-fuses the FOD).
  • Preserve hoisted resolution. Carry bunfig.toml (linker=hoisted) into the agent build and keep the @kolu hydration, or the import graph won’t load at runtime on the remote.
  • Pure eval, no IFD. The fileset must be static literal paths; drvPath stays a pure string interpolation so agentDrvBySystem cross-arch eval survives.
  • Drop the extra install guards (server/main.ts, dist/index.html) in the scoped derivation; keep only agent/main.ts.
  • CI fan-out. If drishtiAgentBuilt is exposed in flake packages, devour-flake builds it on all 3 systems — confirm intended and that the agent-only guard holds on darwin.
  • Verify with --no-eval-cache on a committed tree — see §2.
